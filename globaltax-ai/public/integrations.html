How a company onboards

Step 1 · Pick a primary country

The country choice determines which rule engine runs (PF/ESI/PT for India, FICA/FUTA for US, GPSSA/EOS for UAE, GOSI/EOS for KSA, SI for Egypt). Multi-country tenants get one engine per country.

Step 2 · Choose a data path

CSV for fastest pilot · API for production · Connector when your HRIS is on the supported list. Most pilots start with CSV, graduate to API.

Step 3 · Map your fields

The wizard previews your data, auto-suggests mappings against our schema, and flags missing required fields. Save the mapping as a template per business unit.

Step 4 · Run a dry-run period

One month of historical data is enough to validate the engine matches your existing payroll output to the rupee/dollar. Differences highlight either rule-file gaps or input-data issues — both fixable in-product.

Step 5 · Go live + subscribe to webhooks

Once dry-run reconciles, switch to live. Webhooks push every payrun to your accounting + audit systems so the rule engine becomes the system of record for compliance.

REST API — quick reference

Base URL: https://api.smvps.dev/v1. Auth: OAuth 2.0 client-credentials. Content-Type: application/json.

POST /v1/employees:bulk

// Request — upsert your roster (max 1,000 per call)
{
  "company_id": "acme-in",
  "country": "IN",
  "employees": [
    {
      "id": "EMP-001",
      "name": "Priya Sharma",
      "basic": 50000,
      "hra": 25000,
      "state": "KA",
      "regime": "new",
      "doj": "2022-01-15"
    }
  ]
}

POST /v1/payroll/2026-04:run

// Submit a monthly payroll run; engine returns computed deductions per employee
{
  "company_id": "acme-in",
  "runs": [
    { "id": "EMP-001", "lop": 0, "bonus": 0 }
  ]
}

// Response
{
  "month": "2026-04",
  "computed": [
    {
      "id": "EMP-001",
      "gross": 85000,
      "deductions": { "employeePF": 1800, "pt": 200, "tds": 12500 },
      "netPay": 70500,
      "employerCost": 94310
    }
  ]
}

GET /v1/calendar?country=IN&from=2026-04-01&to=2026-12-31

// Returns the same compliance calendar shown on /calendar
{
  "country": "IN",
  "events": [
    {
      "date": "2026-04-15",
      "title": "EPF ECR upload & payment — Mar 2026",
      "category": "pf",
      "act": "EPF Scheme 1952, Para 38",
      "penalty": "Damages 5–25% u/s 14B + Interest 12% u/s 7Q"
    }
  ]
}

Webhook payload

Every webhook POST is HMAC-SHA256 signed (header X-SMVPS-Signature). Retry policy: exponential back-off up to 24 hours.

// POST to your_endpoint
{
  "event": "filing.due_soon",
  "timestamp": "2026-04-08T10:00:00Z",
  "company_id": "acme-in",
  "data": {
    "date": "2026-04-15",
    "title": "EPF ECR upload & payment — Mar 2026",
    "days_until_due": 7,
    "category": "pf",
    "penalty_if_missed": "5–25% damages + 12% interest"
  }
}

Security & compliance posture

  • Data residency — Indian / US / EU regions with isolated tenants. Data never crosses regions without explicit consent.
  • Encryption — TLS 1.3 in transit, AES-256 at rest. Field-level encryption for PII (name, salary, ID).
  • Rule integrity — every statutory rule file is checksummed; rule changes are versioned and signed by SMVPS Pvt Ltd.
  • Audit trail — every payrun, every rule edit, every API call is logged immutably. Exportable for SOC-2 / ISO 27001 audits.
  • Right to delete — full tenant data export + deletion within 30 days of request, GDPR / DPDP Act 2023 compliant.
